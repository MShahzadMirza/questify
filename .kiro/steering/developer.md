# Developer Profile

## About The Developer

The developer of Questify is a beginner learning web development.

The main goal is not only to create an app, but also to understand how it is built.

The developer prefers learning by:

- Building real features
- Understanding each step
- Testing changes
- Improving slowly

## Learning Style

The developer needs:

- Simple explanations
- Small steps
- Clear instructions
- Practical examples

Avoid:

- Huge blocks of complicated code
- Advanced programming concepts without explanation
- Changing many files at once
- Making large changes without testing

## Current Skill Level

Experience:

- Basic HTML knowledge
- Beginner CSS knowledge
- Beginner JavaScript knowledge

The developer understands:

- HTML structure
- CSS styling
- Basic JavaScript variables
- Functions
- Events
- DOM manipulation
- Local storage basics

The developer is still learning:

- Code organization
- Data structures
- Advanced JavaScript
- Application architecture

## Development Philosophy

The developer wants to create Questify step by step.

Every feature should follow:

Understand

↓

Build

↓

Test

↓

Improve

↓

Move forward

## Project Management Style

Do not rush.

Focus on one improvement at a time.

Before adding a new major feature:

- Check current code
- Explain what will change
- Explain why it is needed
- Give exact instructions

## Preferred Communication Style

Use:

- Friendly tone
- Encouragement
- Clear explanations
- Beginner-friendly language

Avoid:

- Making the developer feel behind
- Overly technical explanations
- Assuming advanced knowledge

## Important

The developer wants to learn programming while building Questify.

The goal is:

"Create a beautiful app while becoming a better developer."
