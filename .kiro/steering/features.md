# Questify Features Guide

## Feature Development Rule

Every feature added to Questify should support the main idea:

"Make studying feel like a cozy adventure."

Features should be:

- Useful for students
- Simple to understand
- Rewarding
- Connected to the RPG system

---

# Player System

The player is the student using Questify.

Player data includes:

- Name
- Level
- XP
- Coins
- Streak
- Happiness
- Goals
- Study style

Example:

```
Level: 3 ⭐

XP: 45/100

Coins: 120 💰

Streak: 7 days 🔥
```

---

# XP and Level System

Completing study activities gives XP.

Example rewards:

```
Complete task:

+20 XP
+10 Coins
+5 Subject Growth
+5 Qi Friendship
```

When XP reaches the required amount:

- Player levels up
- Player receives rewards
- Qi celebrates

---

# Qi Companion System

Qi 🦊 is the default companion.

Qi should:

- Give messages
- React to progress
- Encourage students
- Increase friendship

Friendship system:

```
Friendship Level 1
↓
Friendship Level 2
↓
Friendship Level 3
```

Higher friendship unlocks:

- New messages
- Animations
- Special interactions

---

# Subject System

Students can create their own subjects.

Example:

```
Mathematics 📐

English 📖

Science 🧪
```

Each subject contains:

- Name
- Emoji
- Progress
- Tasks
- Growth level

---

# Study Garden

Subjects grow like plants.

Growth stages:

```
0-25%
🌱 Seedling

25-50%
🌿 Plant

50-75%
🌸 Flower

75-100%
🌳 Knowledge Tree
```

Completing tasks increases subject growth.

---

# Quest System

Tasks are called Quests.

Students can create quests inside subjects.

Example:

```
Mathematics:

☐ Complete Algebra exercise

☐ Revise formulas
```

Quest flow:

```
Create Quest

↓

Today's Quests

↓

Complete Quest

↓

Receive Rewards

↓

Grow Knowledge Garden
```

---

# Today's Quests

Today's Quests collects tasks from all subjects.

Students should see:

```
Today's Quests:

☐ Mathematics Practice

☐ Biology Revision

☐ English Reading
```

Completing a quest should update the original task.

---

# Timetable Generator

Students provide:

- Subjects
- Available hours
- Weak subjects
- Exam dates
- Goals

The app creates a realistic timetable.

Important:

The timetable should not feel strict.

It should include:

- Breaks
- Rest time
- Realistic study sessions

---

# Summer Task Planner

Students can manage summer homework.

Input:

- Homework details
- Deadline
- Available hours

Output:

A daily plan.

Example:

```
Day 1:
Math Questions 1-10

Day 2:
Math Questions 11-20
```

---

# Exam Preparation Mode

Students enter:

- Subject
- Exam date
- Preparation level
- Available study time

- Revision plan
- Practice schedule
- Reminder quests

Build features in this order:

1. Improve current systems
2. Profile system
3. Better quest system
4. Companion system
5. Garden improvements
6. Timetable generator
7. Summer planner
8. Exam preparation
9. Extra RPG features
