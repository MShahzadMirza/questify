# Questify Development Process

## Main Development Philosophy

Questify should be built slowly and carefully.

The developer is learning while creating the app.

The priority is:

Understanding > Speed

A smaller working feature is better than a large unfinished feature.

---

# Development Cycle

Every new feature should follow this process:

## Step 1: Understand

Before writing code:

Explain:

- What the feature does
- Why it is needed
- How it connects to Questify

Do not start coding immediately.

---

## Step 2: Plan

Break the feature into small parts.

Example:

Profile System:

Part 1:
Create profile section in HTML

Part 2:
Add profile data in JavaScript

Part 3:
Connect data to screen

Part 4:
Save profile using local storage

---

## Step 3: Build

Add the smallest working version first.

Avoid adding:

- Extra animations
- Advanced features
- Complex systems

until the basic version works.

---

## Step 4: Test

After every change:

Check:

- Does the app open?
- Does the button work?
- Does the data save?
- Are there console errors?

Fix problems before continuing.

---

## Step 5: Improve

After the feature works:

Improve:

- Design
- User experience
- Code quality
- Small details

---

# Coding Approach

Use the existing project structure:

```
index.html

style.css

script.js
```

Do not create unnecessary files.

The developer prefers a simple structure that is easy to understand.

---

# Teaching Approach

When helping the developer:

Always provide:

1. Explanation
2. File name
3. Exact location
4. Code
5. How to test it

Example:

"Add this inside script.js below the Player Data section."

---

# Avoid Overwhelming Changes

Do not:

- Rewrite the whole application
- Replace working code without reason
- Add many systems together
- Give thousands of lines of code

Prefer:

Small improvements that build on the current project.

---

# Debugging Process

When an error appears:

Follow:

1. Read the error message
2. Find the related code
3. Explain the problem
4. Fix the smallest possible thing
5. Test again

---

# Current Development Focus

The current goal is:

Improve the existing Questify foundation.

Before adding large features:

- Clean current systems
- Make rewards consistent
- Improve saving
- Improve task handling

---

# Success Definition

Questify is successful when:

- Students enjoy using it
- The experience feels cozy
- The developer understands the code
- Features work smoothly
